# 公開手順（GitHub Pages・無料・5分毎自動更新）

このフォルダの中身をそのまま公開します。サーバー不要・無料です。
所要 15〜20分。**リポジトリは必ず「Public（公開）」で作成**してください（Privateだと自動更新が無料枠を超えます）。

---

## 手順

### 1. GitHubアカウントを用意
https://github.com/ で無料登録（既にあれば不要）。

### 2. 新しいリポジトリを作成
1. 右上「＋」→ **New repository**
2. Repository name：例）`tokyowan-schedule`
3. **Public** を選択（重要）
4. 「Create repository」

### 3. ファイルをアップロード
1. 作成直後の画面の **「uploading an existing file」** をクリック
   （既にファイルがある場合は「Add file → Upload files」）
2. このフォルダ（`tokyo-bay-schedule`）の中身を**すべて**ドラッグ＆ドロップ
   - `index.html` / `data.json` / `scrape.js` / `README.md` / 各ガイド`.md`
   - **`.github` フォルダ（中の `workflows/update.yml`）も必ず含める**（自動更新に必須）
   - ※`.github` が見当たらない場合は、下の「補足A」で手動作成
3. 下の「Commit changes」を押す

### 4. GitHub Pages を有効化（公開URL発行）
1. リポジトリの **Settings → Pages**
2. Source：**Deploy from a branch**
3. Branch：**main** ／ フォルダ：**/ (root)** → **Save**
4. 数分後、上部に `https://ユーザー名.github.io/リポジトリ名/` が表示されます
   → これが**スマホで開くURL**。ホーム画面に追加すればアプリのように使えます。

### 5. 自動更新（5分毎）を有効化
1. **Settings → Actions → General**
2. 最下部 **Workflow permissions** を **Read and write permissions** にして **Save**
3. **Actions** タブ → 左「**update-schedule**」→ 右「**Run workflow**」で初回実行
   （以降は5分毎に自動でデータ更新。混雑時は多少遅れることあり）

### 6. 失敗したら気づけるように（推奨）
`運用ガイド_失敗メール通知.md` の手順で、GitHubの失敗通知メールをオンにしてください。

---

## 常に最新にする仕組み（設定済み）
- **自動**：5分毎に最新データへ更新（GitHub Actions／`update.yml`）。
- **手動**：画面右上の **⚓ ボタン**でいつでも即・最新取得（CDNキャッシュ回避つき）。
- **保険**：取得が2時間以上途絶えると画面上部に⚠バナー。取得失敗時も直前データを表示（白画面にならない）。

---

## 補足A：`.github/workflows/update.yml` を手動で作る場合
Webアップロードで `.github` が入らなかったとき：
1. リポジトリ「Add file → Create new file」
2. ファイル名に **`.github/workflows/update.yml`** と入力（スラッシュでフォルダになる）
3. このフォルダの `update.yml` の中身をコピペして Commit

## 補足B：更新頻度を変えたい
`.github/workflows/update.yml` の `cron: "*/5 * * * *"` を編集（`*/10`＝10分毎 など）。

## 補足C：公開範囲について
- Public リポジトリでも、**コードとデータが見えるだけ**で、閲覧者があなたのアカウントを操作することはできません。
- 本サイトが表示するのは、海上保安庁・気象庁・東京都の**公開情報**です。
