# ③ トラブル対処メモ（症状 → 原因 → 対処）

困ったときはこの表を上から確認してください。
**「自分で直せないときは、症状＋GitHub Actionsの赤いログを Claude に貼る」**のが基本です。ほとんどは短時間で直せます。

> 大前提：本サイトは「取得に失敗しても直前のデータを表示」する設計です。
> **いきなり真っ白なエラー画面になることは基本的にありません。**

---

## 1. 予定表が「古い／空っぽ」

- **画面の症状**：上部に黄色い「⚠ データが更新されていません」バナー。更新時刻が進まない。
- **主な原因**：自動更新（GitHub Actions）が失敗している。
- **確認**：リポジトリ → **Actions** タブ → 赤い ✗ の回のログを見る。
- **対処**：
  1. まず **Actions → update-schedule → Run workflow** で手動再実行。直れば一時的な不調。
  2. 直らない場合、ログの赤い部分を **Claude に貼る**（取得元サイトの仕様変更が濃厚。`scrape.js` の解析部分を修正します）。

---

## 2. 天気・気温・風などの「現況パネル」が「--」表示

- **原因**：気象庁JSON／観測ページの一時的な不調、または**深夜で当日の最高/最低気温データが未提供**の時間帯。
- **対処**：多くは**自動で回復**します。半日以上ずっと「--」なら Claude に連絡。
- 補足：現況パネルが取得失敗しても、**予定表そのものは正常に表示**されます（独立処理）。

---

## 3. 潮汐（満潮・干潮）が出ない

- **原因**：気象庁 潮位表ファイル（東京 TK）の取得失敗。特に**年替わり**でファイルのURL年号が変わるタイミング。
- **対処**：`scrape.js` は「現在の年」を自動で使う作りですが、年始に該当ファイルが未公開だと出ないことがあります。数日で解消。続くようなら Claude に連絡。

---

## 4. 「係船（東京港港湾情報システム）」の船が出ない

- **原因**：東京港港湾情報システムの**フォーム／検索手順（POSTパラメータ・セッション）の変更**。
- **対処**：Claude に連絡（`scrape.js` の `fetchPortRows` を修正）。
  ※東航路・西航路（海保）とは独立なので、係船だけ出なくても東西の予定は表示されます。

---

## 5. 船名リンク（MarineTraffic）で目的の船に飛ばない

- **原因**：MMSI対応表に無い船は「船名のローマ字検索」になるため、候補から選ぶ必要があります。日本語船名はヒットしにくいことがあります。
- **対処**：`index.html` の `SHIP_MMSI_RAW` に **「'船名':'MMSI番号'」を1行追加**すれば直リンク化されます。分からなければ Claude に船名を伝えてください（調べて追加します）。

---

## 6. 万一、画面がほぼ真っ白／表示が崩れた

- まず**再読み込み**（右上 ⚓ ボタン、またはブラウザの更新）。スマホなら一度タブを閉じて開き直す。
- キャッシュが原因のことがあるため、**時間を置いて**再アクセス。
- それでも直らなければ、**スマホのブラウザで表示されているメッセージ**や、可能なら**エラー内容**を Claude に伝える。
  （表示生成でエラーが出ても、ヘッダー・現況・直近データは残るよう作ってあります）

---

## 7. 手動でいますぐ最新化したい

- サイト右上の **⚓ ボタン**をタップ → 最新の `data.json` を再取得します。
- 元データ自体を取り直したいときは、GitHub → **Actions → update-schedule → Run workflow**。

---

## Claude に相談するときのコツ
- 「いつから・どの部分が・どうなったか」＋**Actionsの赤いログ**または**画面のスクショ/文言**を貼る。
- これだけで原因特定→修正がスムーズです。
